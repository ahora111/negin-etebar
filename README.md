# وام‌یار — مدیریت مشتریان، پورسانت و محاسبه وام (PWA + Firebase)

اپلیکیشن وب پیشرونده (PWA) برای مدیریت مشتریان، جذب تلفنی، پورسانت‌ها، خرید امتیاز، گزارش سود و **محاسبه وام با فرمول‌های قابل‌تنظیم**.
داده‌ها هم **محلی روی دستگاه** (`localStorage`، برای کارکرد کامل آفلاین) و هم در **Firebase Firestore** (برای همگام‌سازی بین همه‌ی کاربران و دستگاه‌ها) ذخیره می‌شوند.

## ✨ امکانات اصلی

- **مدیریت مشتریان**: ثبت، ویرایش، حذف، فیلتر و جستجو
- **جذب تلفنی**: ثبت لید، پیگیری، تطبیق خودکار با مشتری حضوری
- **پورسانت‌ها**: محاسبه خودکار (درصدی) یا دستی، پرداخت پورسانت
- **گزارش مالی و سود**: محاسبه دقیق سود با تشخیص پرونده‌های ناقص
- **محاسبه وام**: فرمول‌های قابل‌تنظیم برای هر بانک/طرح
- **گفتگوی گروهی**: بین کاربران (با اجازه مدیر)
- **تاریخچه فعالیت‌ها**: ثبت همه‌ی تغییرات مهم
- **گزارش تحلیلی**: نمودارهای قابل‌فهم
- **خروجی اکسل و پشتیبان JSON**
- **کارکرد کامل آفلاین** + همگام‌سازی ابری

## 🚀 راه‌اندازی سریع

### مرحله ۱: ساخت پروژه Firebase
1. به [console.firebase.google.com](https://console.firebase.google.com) بروید و یک پروژه‌ی جدید بسازید (پلن رایگان Spark کافی است).
2. یک اپ «وب» (</>) به پروژه اضافه کنید؛ مقادیر `firebaseConfig` را کپی کنید.
3. آن مقادیر را داخل فایل `js/firebase-config.js` جای‌گزین کنید.

### مرحله ۲: فعال‌سازی Authentication
1. از منوی **Build > Authentication > Sign-in method**
2. روش **Anonymous** را فعال کنید
   - این برای کارکرد همگام‌سازی ضروری است
   - ورود واقعی کاربران با نام‌کاربری/رمز داخل خود اپ انجام می‌شود

### مرحله ۳: ساخت Firestore Database
1. از منوی **Build > Firestore Database**
2. روی **Create database** بزنید (حالت Production)
3. به تب **Rules** بروید
4. محتوای فایل `firestore.rules` (از این پروژه) را کپی و جای‌گزین کنید
5. روی **Publish** بزنید

### مرحله ۴: آپلود فایل‌ها
1. تمام فایل‌های این پوشه را در گیت‌هاب پوش کنید (شامل `js/firebase-config.js` با مقادیر واقعی)
2. `Settings > Pages`: منبع را روی شاخه `main` و پوشه `/root` بگذارید
3. آدرسی مثل `https://username.github.io/repo-name/` فعال می‌شود

### مرحله ۵: ورود اول
- نام کاربری: `admin` / رمز عبور: `admin123`
- **بلافاصله بعد از ورود اول، رمز را عوض کنید**

## 🏦 مدیریت و محاسبه وام

### پنل مدیریت محصولات وام (فقط مدیر)
از منوی «🏦 مدیریت محصولات وام» قابل دسترس است. مدیر می‌تواند برای هر بانک/طرح:
- نام بانک، نام طرح، تعداد اقساط، توضیحات
- فرمول قسط اول، سایر اقساط، مبلغ قابل دریافت (قدرت خرید)
- لیست اقساط خاص (مثلاً قسط سیزدهم)
- لیست ثابت‌ها (درصدها و اعداد)
- فرمول معکوس (اختیاری)
- بایگانی محصول

### بخش محاسبه وام (همه نقش‌ها)
از منوی «🧮 محاسبه وام»:
1. انتخاب بانک/طرح
2. انتخاب حالت: مبلغ وام یا قدرت خرید
3. وارد کردن مبلغ → محاسبه تمام اقساط و مبالغ

### فرمول‌نویسی
فرمول‌ها با موتور امن (بدون `eval`) اجرا می‌شوند. متغیرهای لاتین کوتاه:

| متغیر | معنی |
|-------|------|
| `L` | مبلغ وام |
| `N` | تعداد کل اقساط |
| `I` | شماره قسط فعلی (از ۱) |
| `R` | مانده وام قبل از این قسط |
| `P` | قدرت خرید (فقط در فرمول معکوس) |
| `c.X` | ثابت تعریف‌شده توسط مدیر |

**سینتکس ویژه:**
- `4%` → معادل `4/100`
- `Math_round(x)` → گرد کردن
- `Math_ceil(x)` → سقف (بالا گرد)
- `Math_floor(x)` → کف (پایین گرد)
- اعداد فارسی هم پشتیبانی می‌شوند

**مثال:**
```
L * 4%              → ۴٪ مبلغ وام
L / (N - 2)         → مبلغ وام تقسیم بر (کل اقساط منهای اقساط ویژه)
L - L * 40%         → مبلغ وام منهای ۴۰٪
R * 4%              → ۴٪ مانده وام
```

### نمونه: بانک مهر ایران (پیش‌فرض)
- تعداد اقساط: ۲۴ ماه
- قسط اول: `L * 4%` → ۴٬۰۰۰٬۰۰۰ تومان
- قسط سیزدهم: `R * 4%` → ۲٬۰۰۰٬۰۰۰ تومان
- سایر اقساط: `L / (N - 2)` → ۴٬۵۴۵٬۴۵۵ تومان
- قدرت خرید: `L - L * 40%` → ۶۰٬۰۰۰٬۰۰۰ تومان

## 💰 گزارش مالی و سود

### فرمول محاسبه سود
```
مسیر وجه نقد:
سود = دریافتی خدمات وام − (هزینه خرید امتیاز + پورسانت‌ها)

مسیر خرید کالا:
سود = (مبلغ وام برداشت‌شده + پیش‌پرداخت) − (خرید امتیاز + خرید کالا + پورسانت‌ها)
```

### منطق پورسانت
- اگه کاربر مرتبط (جذب‌کننده/کارشناس) **مشخص** باشه → پورسانت از سود کسر میشه
- اگه کاربر مرتبط **نامشخص** باشه → پورسانت = **۰** لحاظ میشه (سود پرونده همچنان محاسبه میشه)
- پرونده‌های ناقص (بدون خرید امتیاز، serviceFee و...) در کادر زرد نمایش داده میشوند

### پرداخت پورسانت کاربران
زیر گزارش مالی، برای هر کاربر: کل پورسانت، پرداختی، مانده. ثبت پرداخت با کسر از مانده.

## 🔐 امنیت Firestore

### قوانین فعلی (`firestore.rules`)
- فقط کاربران واردشده (Anonymous) دسترسی خواندن/نوشتن دارند
- Collection‌های مجاز صراحتاً فهرست شده‌اند
- `auditLogs` غیرقابل ویرایش/حذف
- سایر مسیرها بسته‌اند

### محدودیت مهم
این اپ از ورود ناشناس (Anonymous Auth) استفاده می‌کند. تفکیک نقش‌ها (مدیر/کارشناس/جذب‌کننده) در سطح اپلیکیشن اعمال می‌شود، نه در Firestore Rules. برای امنیت کامل باید به Firebase Authentication واقعی مهاجرت کنید.

### فعال‌سازی App Check (توصیه‌شده)
1. **Build > App Check** > ثبت اپ وب
2. انتخاب **reCAPTCHA v3**
3. قرار دادن site key در `js/firebase-config.js`
4. بعد از چند روز، **Enforce** را فعال کنید

## 🔄 همگام‌سازی ابری

### چرا داده‌ی دیگران پاک نمی‌شود؟
- هر رکورد یک **سند مستقل** در Firestore است
- هر بار ذخیره، فقط همان سند با `setDoc({merge:true})` بروزرسانی می‌شود
- ادغام بر اساس `updatedAt` (رکورد جدیدتر برنده)
- حذف فقط با عمل صریح مدیر

### رفتار آفلاین
- اپ کاملاً آفلاین کار می‌کند (Service Worker + localStorage)
- Firestore SDK تغییرات را در صف می‌گذارد و هنگام اتصال ارسال می‌کند
- وضعیت اتصال: «☁️ متصل...» / «📴 آفلاین...»

### جلوگیری از تکرار admin پیش‌فرض
- هر دستگاه جدید یک admin موقت (`isSeedDefault`) می‌سازد
- وقتی admin واقعی از cloud می‌رسد، admin موقت حذف می‌شود
- admin موقت هرگز به cloud پوش نمی‌شود
- duplicate های قدینی به‌صورت خودکار پاک می‌شوند

## 🔧 نکات فنی

### تکنولوژی‌ها
- **Frontend**: HTML, CSS, JavaScript (vanilla، بدون فریم‌ورک)
- **Backend**: Firebase Firestore (بدون سرور اختصاصی)
- **Auth**: Anonymous Auth + سیستم داخلی نام‌کاربری/رمز
- **PWA**: Service Worker + Cache API
- **نمودار**: Chart.js (از CDN)
- **اکسل**: SheetJS (از CDN)

### امنیت رمز عبور
- رمزها با PBKDF2-HMAC-SHA256 (۶۰۰٬۰۰۰ تکرار) ذخیره می‌شوند
- ذخیره‌سازی به‌صورت `pbkdf2$iterations$salt$hash`
- ارتقاء خودکار هش‌های قدیمی هنگام ورود

### اعداد فارسی
- اعداد فارسی/عربی در فرمول‌ها و فیلدها پشتیبانی می‌شوند
- تبدیل خودکار به انگلیسی قبل از پردازش

### کش Service Worker
- فایل‌های اصلی: network-first (برای بروزرسانی سریع)
- فایل‌های خارجی (Firebase, Chart.js): cache-first
- نسخه‌بندی با `CACHE_NAME`
